Simplifying Enterprise Data Management and API Development

ChillSharp is a lightweight .NET library that sits on top of Entity Framework Core and turns an existing data model into a fully working REST API with almost no setup.

Instead of writing controllers, services, and endless DTO mappings, ChillSharp exposes your entities through a clean, DTO-based API layer that supports querying, creation, updates, and deletion out of the box.

You bring the DbContext. ChillSharp does the boring parts.

How to build your API server in minutes

      
using ChillSharp;
using ChillSharp.Dto;
using Microsoft.EntityFrameworkCore;

var builder = WebApplication.CreateBuilder(args);

// Assume you already have your DbContext (e.g., AppDbContext) configured
builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(builder.Configuration.GetConnectionString("DefaultConnection")));

// Add ChillSharp API using your existing DbContext
builder.Services.AddChillApi<AppDbContext>();

var app = builder.Build();

// Map ChillSharp API endpoints automatically
app.MapChillApi();

app.Run();

‚úÖ How It Works

  • Existing DbContext
    ChillSharp uses your already-configured EF Core DbContext. No model duplication.
  • Plug-and-Play API
    AddChillApi<TContext>() registers everything needed to expose your data safely and consistently.
  • Automatic Endpoints
    MapChillApi() generates REST endpoints for querying, creating, updating, and deleting entities.
  • DTO-First by Default
    Entities are exposed as ChillDtoEntity, giving you web-friendly JSON without leaking EF internals.

üöÄ What you get

Within less than 10 lines, your database is instantly available through REST endpoints:

  • POST /api/chill/query
    Execute dynamic queries
  • POST /api/chill/create
    Create a new entity
  • POST /api/chill/update
    Update an entity
  • POST /api/chill/delete
    Delete an entity
  • POST /api/chill/chunk
    Executes a chunk of operations in a transaction

All without writing a single controller. Just plug your DbContext and go.
