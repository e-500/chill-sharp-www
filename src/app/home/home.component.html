What it is

From database model to product API to agent capability

ChillSharp is for teams that already have an EF Core model and want to move faster without letting backend plumbing sprawl across controllers, DTO mappers, duplicated CRUD endpoints, and one-off query contracts. Your business entities and queries remain the center of the system; ChillSharp provides the runtime surface.

The same host can serve application clients through REST and AI clients through Model Context Protocol. That makes the product useful for human-facing apps, internal tools, admin workflows, and controlled agent-assisted operations over real business data.

MCP server

An AI can inspect the model before sending a query

ChillSharp.Mcp registers a Model Context Protocol server backed by your ChillSharp context. Agents connect to `/api/chill-mcp`, list the schemas you have enabled, inspect full schema and property descriptions, and then call the exact operation they need.

The intended flow is deliberate: discover schemas, read descriptions, choose a query or entity, validate payloads when needed, then execute the operation under the authenticated user's permissions.

Describe the business model

Use clear Chill type names, useful labels, and strong entity-level `MCPDescription` text so the agent knows what each object represents.

Describe every exposed property

Property descriptions explain meaning, allowed values, units, references, and whether a field is a status, code, lookup, or free-text value.

Prefer focused queries

Purpose-built query types are easier for AI to use reliably than a single broad query with many vague optional inputs.

Keep permissions aligned

MCP-enabled schemas, query visibility, auth permissions, and API-key limitations should describe the same intended access boundary.

Host setup

Add ChillSharp to an ASP.NET Core host

A ChillSharp host starts from your `DbContext`. The API registration can protect the REST and MCP surfaces, while the MCP endpoint remains a separate agent transport at `/api/chill-mcp`. 

using ChillSharp.Api;
using Microsoft.EntityFrameworkCore;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlite("Data Source=app.db"));

builder.Services.AddChillApi<AppDbContext>(options =>
{
    options.ProtectedApi = true;
});

var app = builder.Build();

app.UseAuthentication();
app.UseAuthorization();
app.MapChillApi();
app.Run();

Context shape

Prepare the DbContext for schema and AI consumption

The context inherits `DbContext`, implements ChillSharp context interfaces, and adds the Chill schema model. Entities and queries opt into the AI-facing surface with `EnableMCP = true`. 

public class AppDbContext :
    DbContext,
    IChillContext,
    IChillSchemaDbContext
{
    public string GetChillTypePrefix() => "MyCompany.Product.Data";
    public string GetPrimaryCultureName() => "en-US";
    public string GetSecondaryCultureName() => "it-IT";
    public string GetCurrentUserName() => Environment.UserName;

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);
        modelBuilder.AddChillSchemaModel();
    }
}

AI-assisted development

Less generated infrastructure, more reviewable product logic

ChillSharp helps AI-assisted development by centralizing repetitive API mechanics into a model-driven runtime. Instead of asking an AI tool to keep controllers, DTOs, mappers, validators, route contracts, and CRUD behavior in sync, you can usually ask it to change one entity, one query, one validation rule, or one permission definition.

That reduces accidental interface drift, keeps endpoint behavior uniform, lowers repository context needs, and makes human review focus on business rules rather than repeated plumbing.
